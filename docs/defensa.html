Footprint Mappa Report Builder

Generador de informes PDF de huella de carbono OCF (ISO 14064-1) y PCF (ISO 14067) a partir de CSV. Path A sólido evolucionado a Path B configurable, con un ángulo central de genericidad. Cada afirmación técnica de este documento apunta a archivo:línea con un fragmento verbatim del código.

Stack: Next.js 16 · React 19 · Tailwind v4 · shadcn/ui · Puppeteer/Chromium · Playwright E2E Render: un único componente HtmlReport → preview A4 + PDF Backend: solo POST /api/pdf

export function buildReport(rows, fileName = "Uploaded CSV", options = {}) {
  const kind = detectReportType(rows);

  if (kind === "ocf") {
    // OCF uses an organisational boundary; the PCF boundary option is ignored.
    return buildOcfReport(rows, fileName);

  }

  throw new Error(

export function applyColumnMapping(rows, mapping) {
  const entries = Object.entries(mapping || {}).filter(([, source]) => source);
  return (rows || []).map((row) => {
    const next = { ...row };
    for (const [canonical, source] of entries) {
      next[canonical] = row[source];

    return next;
  });

Mira la primera línea no vacía (la cabecera, sin números → inequívoca), cuenta cada candidato y elige el mayor; default ,.

const DELIMITER_CANDIDATES = [",", ";", "\t"];

function detectDelimiter(text) {
  const headerLine = text.split(/\r\n|\r|\n/).find((line) => line.trim() !== "");
  if (!headerLine) {
    return ",";
  }
  let best = ",";
  let bestCount = 0;
  for (const candidate of DELIMITER_CANDIDATES) {
    const count = headerLine.split(candidate).length - 1;
    if (count > bestCount) {
      best = candidate;
      bestCount = count;
    }
  }
  return best;
}

Fila Total del OCF, sin depender del idioma

etiqueta flexible → fallback estructural (≈ suma de las demás, ±1%)lib/data/ocf.js:106-138
function findTotalRowIndex(normalizedRows) {
  const byLabel = normalizedRows.findIndex((row) =>
    TOTAL_ROW_LABELS.has(row.entity.trim().toLowerCase()),
  );
  if (byLabel !== -1) {
    return byLabel;
  }
  if (normalizedRows.length < 2) {
    return -1;
  }
  const grandTotal = sumBy(normalizedRows, (row) => row.totalEmissions);
  let match = -1;
  for (let index = 0; index < normalizedRows.length; index += 1) {
    const value = normalizedRows[index].totalEmissions;
    const others = grandTotal - value;
    if (others <= 0) {
      continue;
    }
    if (Math.abs(value - others) <= others * 0.01) {
      if (match !== -1) {
        return -1;
      }
      match = index;
    }
  }
  return match;
}

Avisos por valores no parseables (no se redondea a 0 en silencio)

vacío = 0 sin aviso; no vacío no parseable = 0 + malformedlib/data/numeric.js:8-30
export function parseNumberStrict(value) {
  if (typeof value === "number") {
    return Number.isFinite(value)
      ? { value, malformed: false }
      : { value: 0, malformed: true };
  }
  const raw = String(value ?? "").trim();
  if (raw === "") {
    return { value: 0, malformed: false };
  }
  const normalized = raw
    .replace(/\s/g, "")
    .replace(/%/g, "")
    .replace(",", ".");
  const number = Number(normalized);
  return Number.isFinite(number)
    ? { value: number, malformed: false }
    : { value: 0, malformed: true };
}

export function applyPreset(sections, presetId, kind = "ocf") {
  const preset = getSectionPresets(kind)[presetId];
  if (!preset) {
    return sections;
  }
  const byId = new Map(sections.map((section) => [section.id, section]));
  const orderedBaseSections = preset.sectionIds
    .map((id) => byId.get(id))
    .filter(Boolean)
    .map((section) => ({ ...section, enabled: true }));
  const orderedIds = new Set(preset.sectionIds);
  const disabledBaseSections = sections
    .filter((section) => !section.removable && !orderedIds.has(section.id))
    .map((section) => ({ ...section, enabled: false }));
  const customSections = sections.filter((section) => section.removable);
  return [...orderedBaseSections, ...disabledBaseSections, ...customSections];
}

K Preguntas probables del tribunal + respuestas

¿Por qué Path B y no Path A simple o Path C?

El valor de producto está en la configurabilidad y la genericidad; Path C habría sido débil y arriesgado en demo. El brief dice que un Path A fuerte > Path C débil. Entrego Path B sólido con Path C documentado (README.md:11-18).

¿Por qué Chromium y no @react-pdf/renderer?

Para no mantener dos superficies. Un solo HtmlReport sirve para preview (ReportBuilder.jsx:1128-1136) y PDF (renderHtmlReport.js:49-73).

¿Playwright o Puppeteer?

Ambos son válidos para HTML→PDF. Playwright se mantiene para E2E; el endpoint productivo usa Puppeteer porque puppeteer-core + @sparticuz/chromium-min encaja mejor con Vercel Functions.

¿Qué fallo apareció en Vercel y cómo se resolvió?

Vercel fallaba al descargar PDFs con Cannot find module '/var/task/node_modules/playwright-core/browsers.json'. No era un fallo del HTML ni del modelo del informe: era un problema de empaquetado/runtime de Playwright en la Function serverless. La solución fue conservar la arquitectura HTML→PDF, pero cambiar el launcher productivo a Puppeteer con Chromium serverless.

¿La preview es idéntica al PDF?

Mismo componente, estilos y SVG; la única diferencia es la paginación (cortes finales por Chromium). La preview los aproxima con guías de página (ReportBuilder.jsx:740-767, ReportBuilder.jsx:1137-1143).

¿Y un CSV con ';' o decimales con coma?

Separador autodetectado (parseCsv.js:6-22) y coma decimal → punto (numeric.js:21-24).

¿Qué pasa con un valor no parseable?

Vacío = 0 sin aviso; no vacío no parseable = 0 + warning (numeric.js:8-30, ocf.js:80-86) → toast (ReportBuilder.jsx:87-91, ReportBuilder.jsx:244-247). Nunca redondeo a 0 en silencio.

¿Cómo detectas la fila "Total" sin depender del idioma?

Etiqueta flexible → estructural (≈ suma de las demás, ±1%) → calculado; exclusión por índice (ocf.js:106-138, 167-171).

¿El mapeo es un motor genérico de CSV?

No: solo OCF/PCF y solo al fallar (schema.js:1-3). Cubre columnas renombradas sin sobreingeniería.

¿Por qué cradle-to-gate para Relats?

Componentes intermedios; downstream depende del producto final. Default en pcf.js:104, conmutable.

¿Por qué no sumas los productos?

Unidades funcionales distintas; cifra combinada etiquetada como comparativa (pcf.js:302-311).

¿La app calcula la huella?

No; presenta datos ya calculados. Factores de emisión fuera de alcance (README.md:280).

¿Por qué las gráficas no usan el color del cliente?

Legibilidad y consistencia; chrome vs datos separados (brandTheme.js:1-9, chartTheme.js:5-35).

¿Cómo aseguras un acento legible?

ensureReadable oscurece hasta contraste ≥3:1 (contrast.js:73-86); bestTextOn elige texto. Tests en contrast.test.mjs/brandTheme.test.mjs.

¿Cómo extraes la paleta del logo?

Canvas 64px, buckets, descarte de transparencias/blancos, ranking + distancia mínima (extractPalette.js:52-130). Solo navegador.

¿Dónde entraría la IA y por qué no está?

Endpoint /api/report/assist diseñado, no implementado (pdf-technology-recommendation.md:84-235); evita API keys/coste/indeterminismo en demo.

¿Qué tests tienes?

Unitarios node --test: parseCsv, ocf, pcf, robustness, genericity, contrast, brandTheme (package.json:10); E2E responsive con Playwright en 6 viewports (tests/responsive.e2e.mjs:9-17).

¿Por qué un solo backend y no full-stack?

El único endpoint es POST /api/pdf (route.js:12); CSV se parsea en el navegador y no se persiste (README.md:9).
